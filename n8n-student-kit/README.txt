n8n Student Kit - Інструкція для початківців
============================================

Це набір скриптів для встановлення і запуску self-hosted n8n на Windows.
Після першого налаштування використовуйте лише:
- start.bat (запуск)
- stop.bat (зупинка)
- або GUI-оболонку: start-gui.bat

--------------------------------------------
1) Що підготувати ПЕРЕД встановленням
--------------------------------------------

1.1. Обов'язково:
- Windows 10/11
- Доступ до інтернету
- ngrok-акаунт
- Reserved ngrok domain (наприклад your-name.ngrok-free.dev)
- ngrok Authtoken (саме Authtoken, не API key)

1.2. Файли, які треба покласти у папку tools:
- DockerDesktopInstaller.exe або Docker Desktop Installer.exe
- ngrok.exe (або встановлений ngrok у PATH)

1.3. Опційно (для швидкого/офлайн першого запуску n8n):
- tools\images\n8n-2.15.0.tar
  (або n8n-<N8N_VERSION>.tar, якщо зміните версію у install.bat)

--------------------------------------------
2) Що всередині набору
--------------------------------------------

- install.bat           перша установка / repair / reconfigure
- start.bat             повторний запуск (Docker + n8n + ngrok)
- stop.bat              зупинка n8n і ngrok
- uninstall-local.bat   локальна очистка конфігів (без повного деінсталу Docker)
- start-gui.bat         запуск графічного інтерфейсу
- N8N-Student-Kit-GUI.ps1  GUI-оболонка для кнопок Install/Start/Stop/Cleanup
- compose\docker-compose.yml
- state\                службові файли після установки

--------------------------------------------
2.1) Швидкий старт через GUI
--------------------------------------------

1. Запусти start-gui.bat.
2. Заповни поля domain/token/login/password і натисни Save Config.
3. Натисни Install (Admin) для першого встановлення.
4. Для щоденного використання - кнопки Start / Stop.
5. Якщо не треба авто-відкриття браузера - Start (--no-browser).
6. Хід виконання дивись у live-лог внизу GUI (та у state\gui-*.log файлах).

--------------------------------------------
3) Перший запуск (з нуля)
--------------------------------------------

Важливо:
- Рекомендовано запускати install.bat через CMD з правами адміністратора.
- Якщо запустити без адмін-прав, скрипт спробує підняти UAC автоматично.
- Прапор --elevated службовий, його використовує сам скрипт після UAC-перезапуску.

Кроки:
1. Відкрийте CMD as Administrator.
2. Виконайте:
   cd /d D:\N8N-Workstation\n8n-student-kit
   install.bat
3. Введіть:
   - диск для даних n8n (наприклад D)
   - ваш reserved ngrok domain
   - ngrok authtoken
   - логін для n8n basic auth
   - пароль для n8n basic auth

Після цього інсталер:
1) перевіряє WSL
2) перевіряє/встановлює Docker Desktop
3) налаштовує ngrok token
4) створює state\install.cfg і state\.env
5) чекає Docker
6) готує образ n8n:
   - спочатку docker load з tools\images\n8n-<N8N_VERSION>.tar
   - якщо tar нема, тоді docker pull з Docker Hub
7) запускає n8n (через start.bat)
8) відкриває браузер на https://<NGROK_DOMAIN>/
   (або не відкриває, якщо запускали start.bat --no-browser)

Примітка:
- Якщо на етапі WSL або Docker потрібен reboot, install.bat зупиняється штатно і просить перезапуск ПК.
- Після перезавантаження просто запустіть install.bat ще раз.

--------------------------------------------
4) Повсякденне використання
--------------------------------------------

Запуск:
- start.bat
- Якщо n8n вже запущений, start.bat не падає: пропускає повторний compose up, піднімає/оновлює ngrok і відкриває URL.
- Якщо не треба відкривати браузер автоматично: start.bat --no-browser

Зупинка:
- stop.bat

Зміна параметрів (домен, токен, логін, пароль, диск):
- знову запустіть install.bat
- коли спитає "Використати існуючу конфігурацію?" натисніть N

Локальна очистка:
- uninstall-local.bat видаляє тільки state\install.cfg і state\.env після stop
- uninstall-local.bat НЕ видаляє <обраний_диск>:\n8n-data і Docker Desktop

--------------------------------------------
5) Де зберігаються дані
--------------------------------------------

- Робочі дані n8n: <обраний_диск>:\n8n-data
  (workflow, credentials, налаштування)
- Службовий конфіг кита: n8n-student-kit\state\
- У state\ файлах зберігаються токени/паролі в plain text. Не публікуйте ці файли.

--------------------------------------------
6) Як змінити версію n8n
--------------------------------------------

Версія задається в одному місці:
- install.bat -> N8N_VERSION=2.15.0

Після зміни версії:
1) (опційно) підготуйте tar:
   docker pull n8nio/n8n:<VERSION>
   docker save -o tools\images\n8n-<VERSION>.tar n8nio/n8n:<VERSION>
2) запустіть install.bat
3) після зміни версії обов'язково знову запустіть install.bat, щоб оновився state\.env

--------------------------------------------
7) Типові проблеми і рішення
--------------------------------------------

Проблема: "ngrok процес не запустився"
- перевірте, що токен правильний
- перевірте, що це саме Authtoken з ngrok dashboard

Проблема: "dial tcp [::1]:5678 ... refused"
- n8n контейнер не піднявся або впав
- перевірте:
  docker ps -a
  docker logs n8n --tail 100

Проблема: "container name '/n8n' is already in use"
- лишився старий контейнер
- зупиніть/видаліть його і повторіть start.bat

Проблема: Docker довго стартує
- відкрийте Docker Desktop вручну
- дочекайтеся статусу Running
- повторіть start.bat

Проблема: docker --version працює, а docker info дає помилку про docker_engine
- це означає, що CLI встановлено, але Docker daemon ще не піднявся
- запустіть Docker Desktop і дочекайтесь ready стану
- після цього повторіть install.bat або start.bat

Проблема: install.bat завершився повідомленням про reboot
- це очікуваний сценарій після ввімкнення WSL або встановлення Docker Desktop
- перезавантажте Windows і запустіть install.bat ще раз

--------------------------------------------
8) Швидка перевірка після установки
--------------------------------------------

1. У браузері відкривається https://<NGROK_DOMAIN>/
2. Є basic auth вхід (логін/пароль, який ви задавали)
3. Локально відкривається http://localhost:5678
4. Команда docker ps показує контейнер n8n у статусі Up
